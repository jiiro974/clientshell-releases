Section 01

Quick Start

Get up and running in under two minutes. No external dependencies required.

1

Prerequisites

Go 1.22+ and Docker installed on your system.

2

Build

# Build all binaries
go build ./cmd/clientshell
go build ./cmd/cs-toolbox
go build ./cmd/cs-server
3

First Run

# Start interactive shell
./clientshell

# Connect to a client
./clientshell connect acme-corp

Section 05

Security

Every action is authenticated, authorized, and recorded in a tamper-proof chain.

⛓

Immutable Audit Trail

Every user action is recorded in a SHA-256 hash chain. Each entry includes the hash of the previous entry, creating a tamper-evident, append-only log that can be independently verified.

hash[n] = SHA256(hash[n-1] + action + timestamp + user)
⚿

HMAC-SHA256 Token Auth

All API communication is authenticated using HMAC-SHA256 signed tokens. Tokens are time-limited and bound to the originating session for replay attack prevention.

Authorization: HMAC-SHA256 ts=<epoch> sig=<hex>
⚙

Role-Based Access Control

Four-tier RBAC system enforces least-privilege access across all operations.

Permission Viewer Analyst Admin Super
View sessions ✓ ✓ ✓ ✓
Execute commands ✗ ✓ ✓ ✓
Manage users ✗ ✗ ✓ ✓
Kill sessions ✗ ✗ ✓ ✓
System config ✗ ✗ ✗ ✓
⚖

Bidirectional Audit

Every request carries an X-Request-ID header for full correlation between client and server audit trails. Both sides log independently, enabling cross-verification.

X-Request-ID: <uuid> (correlated across client + server)
🔒

Secrets Never in Clear Text

Secret values are never written to logs, stdout, stderr, or error messages. The secret scanner detects and masks credentials, API keys, tokens, and private keys in all output streams before they reach any sink. Integration with Keeper and Vault ensures secrets are retrieved just-in-time and never persisted locally.

Section 06

API Reference

REST and WebSocket endpoints for programmatic access and integration.

cs-server REST API

GET
/api/health
Server health check. Returns status, uptime, and version.
GET
/api/status
Active sessions, connected clients, and system resource usage.
GET
/api/recordings
List session recordings with metadata, timestamps, and audit hashes.

WebSocket

WS
ws://localhost:8080/ws
Real-time bidirectional channel for shell I/O, events, and session control. Supports binary frames for terminal data and text frames for JSON control messages.

cs-toolbox API

GET
/health
Toolbox service health and loaded tool count.
GET
/tools
List all available tools with names, categories, and parameter schemas.
POST
/exec
Execute a tool by name with parameters. Returns structured JSON results. Requires HMAC-SHA256 authentication.
# Example: Execute a tool via the API
curl -X POST http://localhost:9090/exec \
  -H "Content-Type: application/json" \
  -H "Authorization: HMAC-SHA256 ts=1700000000 sig=a1b2c3..." \
  -d '{"tool": "whois", "params": {"target": "example.com"}}'

New in v2.0

Distributed architecture, system monitoring, SQL engine, and plugin system

cs-query (SQL Engine)

Query system state with SQL-like syntax. 7 virtual tables: processes, connections, listening_ports, users, mounts, interfaces, env.

cs-toolbox query "SELECT pid, name FROM processes WHERE user = 'root'"
cs-toolbox query ".tables"
cs-toolbox query "SELECT * FROM connections WHERE state = 'ESTABLISHED'"

cs-sysmon (Monitoring)

Sysinternals-like tools: process monitor with tree view, autoruns scanner (systemd/launchd/cron), and tcpview with connection diff.

cs-toolbox procmon --tree --sort cpu
cs-toolbox autoruns --new
cs-toolbox tcpview --diff

Distributed Mode

Multi-agent orchestration: broadcast commands to all agents, group by tag, chain with pipelines.

:agents list
:broadcast health system
:agent acme netexplorer --alerts

Network Explorer

Real-time connection monitor with IP reputation scoring (0-100), auto-refresh, and color-coded alerts.

cs-toolbox netexplorer --watch
cs-toolbox netexplorer --reputation
cs-toolbox netexplorer --watch --interval 2

Plugin System

Modular architecture with setup wizard, predefined bundles (minimal/network/soc/full), and auto-upgrade.

cs-toolbox setup --bundle soc
cs-toolbox plugin list
cs-toolbox upgrade --all

Web Capture

Capture web pages via HTTP (no browser needed). Full report with HTML source, headers, TLS info, and title.

cs-toolbox screenshot https://example.com
cs-toolbox screenshot https://10.0.0.1 --insecure
cs-toolbox screenshot url --format headers
